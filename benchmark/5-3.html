What V16.18.4 changed

Path-A entropy stage: RangeCoder (Witten-Neal-Cleary carry-correct) and RangeCoderO1 (order-1 context) added as new candidates alongside Rice/Unary/Split. RangeCoder fired in 100% of TextPipeline path-A picks across the corpus.

UnaryBEP: apex-rank-zero shortcut formats 0x02/0x03 added (1-bit encoding for rank 0 = 47-52% of post-MTF values on text). Encoder picks smallest of {0x00, 0x01, 0x02, 0x03}.

RePair: cost-aware rule selection enabled (RR3). Picks pairs by net byte savings rather than raw frequency. Python-verified +0.5–1.5% on Calgary text files.

Mechanism behind the regression: V16.18.4 tightens the BEP byte-level archive, but the resulting bytes are also closer to Shannon-flat — leaving nothing for the Nibble/Nibble3/Arith wrap layers (4 candidates inside TextCtx) to exploit. On large text, TextCtx with V18 wraps loses ~1-3% of file size compared to V17's "looser BEP + profitable wrap" pattern. The V16.19 hybrid drop addresses this by computing both profiles and picking the smaller per file.
